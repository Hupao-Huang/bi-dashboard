# 数据关联三维图 (data-map 3D) 设计

**日期**: 2026-06-16
**范围**: BI 看板新增一张 3D 数据架构关联图, 仅超管可见
**状态**: 已与跑哥对齐, 待实现

## 目标

给超管一张可旋转的 3D 力导向网络图, 一眼看清「数据从哪来 → 汇成哪些数据 → 喂给哪些看板」的全景架构。纯展示, 不连库, 架构模型人工梳理保证准确。

## 三层节点模型 (高阶总览, ~30 节点)

- **第①层 外部数据源 (source)**: 吉客云 ERP / 用友 YonBIP / 合思费控 / 钉钉 / RPA(Z盘11平台)
- **第②层 数据域 (domain)**: 销售单 / 销售汇总账 / 库存快照 / 调拨 / 运营数据 / 采购委外(用友) / 费控审批 / 商旅订单 / 客服(服务分·评论) / 财务报表 / 原料行情
- **第③层 看板模块 (board)**: 综合看板 / 电商·店铺·货品 / 供应链(采购·库存·快递仓储) / 财务(财报·利润·SKU动销) / 客服看板 / 费控管理 / 合思审批机器人 / 营销看板 / 原料行情 / 销量预测 / AI 智能助手

连线 = 真实数据流向。三层用三种颜色区分。AI 智能助手跨域连多个数据域。

## 技术方案

- **库**: `react-force-graph-3d` (Three.js, 自动 3D 力布局 + 拖转/缩放/高亮)。**React.lazy 懒加载**, 仅本页加载, 不拖累其他页打开速度。
- **备选(否决)**: echarts-gl — 对 echarts 6.0 兼容无保证 + 无自动 3D 布局。
- 节点/连线 = 前端静态 TS 模型 (`src/pages/system/dataMapModel.ts`), 无后端数据接口。

## 组件 / 接入

- 新页 `src/pages/system/DataMap.tsx`: 懒加载 ForceGraph3D, 三层配色, 图例, 悬停显示节点说明, 点节点高亮其上下游连线
- 路由 `/system/data-map`, 挂"系统管理"菜单组
- 权限点 `system.data_map:view` (Type page), **只种给超管**(不进任何普通角色默认权限 → 照 SKU 动销率同款套路, 仅超管可见菜单/页面)
- 后端仅 auth_seed.go 加一个权限点; 启动 seedSuperAdminRolePermissions 自动授超管

## 交互

拖动旋转 / 滚轮缩放 / 悬停节点显示名称+说明 / 点节点高亮相连节点和连线 / 图例标三层颜色

## 验收

- 超管登录能在系统管理见到"数据关联图"菜单, 打开是可旋转 3D 网络球, 节点连线符合真实架构
- 非超管角色看不到菜单、直接访问 /system/data-map 被权限拦
- 其他页面打开速度不受影响(懒加载分包)

## 非目标 (YAGNI)

- 不下钻到具体表 (跑哥选了高阶总览)
- 不连实时库 / 不做编辑功能 (纯只读架构图)
- 不自动从库推导关系 (库无外键, 人工梳理)
