# 小红书乘风看板 设计文档

2026-06-23 / 跑哥确认口径

## 背景

小红书运营看板拆成两个独立菜单(社媒部门下):
- **小红书千帆**：现有「笔记/商品」看板(`XiaohongshuDashboard.tsx` + `/api/xiaohongshu/*`),仅菜单名从"小红书看板"改为"小红书千帆",页面/接口/数据**不动**。
- **小红书乘风**：新建,展示信息流投流数据(`op_xhs_chengfeng_daily`,已扩到 145 业务字段)。

## 一、菜单与路由

- `navigation.tsx`:社媒部门下,现有"小红书看板"label 改为"小红书千帆"(route `/social/xiaohongshu` 不变);新增"小红书乘风"(route `/social/xiaohongshu-chengfeng`),权限位沿用社媒部门 scope(`social`)。
- `App.tsx`:加 `/social/xiaohongshu-chengfeng` → 新组件 `ChengfengDashboard`。

## 二、乘风页面布局(仿千帆笔记页)

1. **筛选**:日期范围(stat_date,默认=最新一天)+ 店铺多选(4 个沃虎店)。
2. **KPI 卡**(顶部):总消费、总展现量、总点击量、7日总支付金额(GMV)、综合ROI(=Σ7日支付金额/Σ消费)。大数字下配 ≈X万 辅助文字(沿用 [[project_wan_hint_ui]] 模式)。
3. **明细表**:按 `note_id` 聚合区间,展示跑哥指定的 108 列(顺序见附录),笔记ID/标题左侧固定列(sticky),其余横向滚动。按 Σ消费 倒序,TOP 200。笔记ID 可复制 + 链接(仅 http)。
4. **行展开趋势**:单笔记按 stat_date 每天 消费(柱) vs 7日支付金额(线),仿现有 `NoteTrend`。新接口 `chengfeng/note-trend`。

## 三、后端接口(`ops_xhs_chengfeng.go`,scope=social)

- `GET /api/xiaohongshu/chengfeng/filters` → `{latestDate, shops}`
- `GET /api/xiaohongshu/chengfeng/list?start&end&shops&note_id_like` → `{kpi, detail, dateRange}`
- `GET /api/xiaohongshu/chengfeng/note-trend?note_id&start&end` → `{trend}`
- 全部 `requireDeptAccess(r,"social")` + `context.WithTimeout 30s`,复用 `xhsCond` 风格拼 shops 条件。

## 四、聚合口径(花钱数据,核心)

按 `note_id` GROUP BY 跨天聚合。**量类直接 SUM;率类禁简单平均,用 Σ分子/NULLIF(Σ分母,0) 加权重算**([[feedback_rate_metric_weighted_not_simple_avg]])。

### 4.1 量类(直接 SUM)
消费、展现量、点击量、新客消耗、新客曝光、新客点击、平台额外补贴金额、补贴撬动全店交易额、点赞、评论、收藏、关注、分享、互动量、行动按钮点击量、截图、保存图片、搜索组件点击量、搜后阅读量、预约人次、直播预约人次、预约触达人群开播曝光/观看/下单人次、直播间观看次数、直播间新增粉丝数量、直播间5s/30s观看次数、直播间评论次数、直播间直接商品访客量/加购量、7日总商品访客量/加购量、视频播放量、视频5s播放量、各类订单量/金额(7日下单/支付、转化时间口径、直接支付、商品直接下单、商品1日支付、直播间直接下单/支付、预约触达人群各口径、店铺新客各口径)。

### 4.2 率类(精确分子分母,口径明确)
| 列 | 公式 |

| 新客点击率 | Σ新客点击 / Σ新客曝光 |

| 新客平均千次展示费用 | Σ新客消耗 / Σ新客曝光 × 1000 |
| 平均互动成本 | Σ消费 / Σ互动量 |
| 视频5s完播率 | Σ视频5s播放量 / Σ视频播放量 |
| 直播间观看成本 | Σ消费 / Σ直播间观看次数 |
| 直播间5s观看成本 | Σ消费 / Σ直播间5s观看次数 |
| 直播间30s观看成本 | Σ消费 / Σ直播间30s观看次数 |
| 直播间直接商品访客量成本 | Σ消费 / Σ直播间直接商品访客量 |
| 直播间直接商品加购量成本 | Σ消费 / Σ直播间直接商品加购量 |

## 五、验证计划(金标准)

1. **单日对拍**:选某一天(如 6-22),聚合结果 vs 该天原始行,量类逐笔对应、率类与原始值一致(单日单笔记聚合=原值)。
2. **多日加权 sanity**:跨多天,ROI/点击率落在各天值区间内,不被小样本日拉偏(加权特征)。
3. **KPI vs 明细一致**:KPI 总消费/总GMV = 明细全量 Σ(后端全口径,前端不 reduce TOP-N,[[feedback_frontend_reduce_with_limit]])。
4. **前端 build + playwright 实测**:乘风菜单可进、表格横滚+固定列正常、行展开趋势出图、千帆改名后照常([[feedback_test_and_verify]])。

## 六、不做(YAGNI)

- 不做乘风的物化预聚合表(数据量小,225/天,实时聚合够快)。
- 不做笔记发布时间筛选(乘风无 note_create_time)。
- 历史 6-1~6-21 数据新字段暂为 0(跑哥让 RPA 补宽表文件后重导),乘风页展示这些日期时长尾列会是 0,正常。
