## 二、算法定义（混合版 = 分而治之）

**核心思想**：不同月份用各自最准的招。

- **春节月（1月、2月）**：`建议 = 该SKU该大区去年同月销量 × 大区增长系数`
  - 大区增长系数 = 该大区近3月销量 ÷ 去年同期3月销量，封顶封底 [0.95, 1.10]
  - **不再叠**季节系数 / 节假日因子 / 趋势（春节销量与平时脱节，多叠反而带偏）
  - 理由：春节就该看去年同期，回测春节月误差从 39% → 13%

- **平淡月（3-12月）**：见下方两个档位二选一

## 三、两个档位（请跑哥二选一）

### 🟢 档位1 — 小改（推荐）
- **平淡月：保持现状不动**（季节版，按 SKU×大区 精细算，对长尾 SKU 稳）
- **只把春节月（1/2月）切成"去年同月×增长"**
- 整体 14.1% → **10.7%**，春节最大偏差 47% → 21%
- 改动小、风险低，**数据才 26 个月、春节就 2 个样本，不值得大动**
- 先治住唯一真翻车的春节，上线看效果，再决定要不要做档位2

### 🟡 档位2 — 大改
- **平淡月也从季节版换成"加权版"**（近3月均 + 同比 + 环比 加权）
- 春节月同样切"去年同月×增长"
- 整体 → **8.3%**，最准
- 但：①改动面大 ②加权版回测是大区合计测的，落到**单个 SKU×大区**时数据稀疏（很多 SKU 一个月就卖几件），可能比合计更跳，**需额外抽验长尾 SKU** ③需新查每个 SKU×大区的"去年同月""上月"数据

> **推荐档位1**：用最小代价（只动春节）拿到大部分收益（14.1→10.7），平淡月保持 SKU 级稳定。等攒够数据、看到效果，再迭代档位2。

## 五、需要跑哥拍板的点

1. **档位1 还是档位2？**（推荐档位1）
2. **春节月要不要 ×增长？**
   - 数据：1月×增长更准（春节确实涨）、2月纯去年同月更准（去年偏高）；整体×增长略好且封顶±10%安全。**推荐：×增长**
3. **营销污染防护**（仅档位2涉及）：
   - 现状季节版自带"某SKU某月波动>30%→用品类中位数替代"的防促销污染机制。
   - 档位1不动平淡月 → 这套**继续生效**，无需操心。
   - 档位2换加权版 → 这套（针对季节系数的）**不再适用**，加权版靠"近3月均平滑+增长clamp"兜底；如担心，可加简化版污染过滤。**档位1可完全回避此问题。**
